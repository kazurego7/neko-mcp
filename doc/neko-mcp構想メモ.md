# neko-mcp 構想メモ

## 目的とゴール
- OpenAI Apps SDK を使って「会話の途中に猫がひょっこり現れる」体験を作る。
- 完成イメージは、AI ⇔ ユーザーの真面目な対話の横で猫が空気をゆるめてくれる世界。

## MVPで試すこと
1. AI応答の途中で猫が画面を横切り、鳴き声が混じる演出を入れる。
2. 「ちょっと休憩したい」と言うと、猫の画像ギャラリーがふわっと開く。
3. ユーザーが猫に話しかけると、しっぽや鳴き方だけでリアクションが返る。

## 今はやらないこと
- 3Dレンダリングや複雑なアニメ制作（まずはシンプルなモーションで十分）。
- 会話ロジックの高度なパーソナライズ。
- 多言語対応や他ペット展開などの横展開。

## アイデア詳細メモ

### 背景
- OpenAIのAppsSDKを使って、なんか面白いことをしたい。
- 会話の中に溶け込ませるなら何がいいだろう？ → 猫じゃね？
- 猫のMCPとかなさそうだし作るべ。

### コンセプト
- AIとの会話の中で、猫がふっと場を和ませる瞬間を作る。
- 猫は喋らずに、猫らしく会話を邪魔してくる。
1. 猫がAIの説明を邪魔してくる  
   - AIの説明が途中で"あっ"と途切れる ⇒ 猫が横切る絵 ⇒ 改めて説明しなおす
   - 下に向かって猫パンチする絵 ⇒ AIが「イタイよ～」と反応する。
2. AIが休憩のために、猫の写真一覧を出してくれる  
   - 猫があくびする絵 ⇒ 猫画像ギャラリーが開く。
3. ユーザーから猫への語り掛けをすると、猫からの反応をAIが説明する
   - A. しっぽと耳だけで反応、ゴロゴロ鳴っている（ポジティブな反応）。  
   - B. チラッと見て、あくび・伸び（無視・無関心）。  
   - C. シャーっと言って逃げる（ネガティブな反応）。

### UX設計の方向性
- 猫は AIの「相棒」ではなく、空間にいるもう一匹の存在。
- 対話の主役はあくまで AI ⇔ ユーザー。
- 猫はその「間（ま）」に割り込んだり、空気を変えたりするだけ。
- 語らず、鳴き声と仕草と間合いだけで空気をコントロール。
